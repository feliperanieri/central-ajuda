1.1 Empresa (parâmetros)

A Empresa Principal é criada pelo Administrador make, mas os dados podem ser alterados pelo usuário que tiver acesso a alterar Empresa (Parâmetros).

Para fazer alteração das informações e parametrizar os dados acesse:
Menu > Administração > Estrutura Organizacional > Empresa (Parâmetros).

Descrição dos parâmetros

Expira a sessão do usuário em
Caso o usuário não navegue pelo sistema durante o tempo estipulado a sessão é expirada e será necessário novo login. Este campo aceita o valor máximo de 60 minutos, é possível para o usuário alterar e reduzir este tempo.
Senha expira
Quando selecionado, a senha do usuário será expirada ao atingir 60 dias de uso. Para reativá-la, será necessário entrar em contato com o Usuário Responsável ou um Usuário com acesso completo ao Cadastro de Pessoas.
Seleção de cliente
Define se a seleção de clientes no sistema será por combo ou lupa
Nome do cadastro de entidades
Define o nome do cadastro de Entidades. Ex.: Clientes. Caso não esteja preenchido assume o padrão Entidade.
Calendário
Define o calendário que será utilizado pela empresa.
url botão sair
Permite o preenchimento de uma URL (Localizador Universal de Recursos), a ser chamada em caso de Logoff (Botão "Sair" do Menu). Ex. URL preenchida: "www.maestro.inf.br", ao sair do sistema pelo menu "Sair" a tela será redirecionada ao site www.maestro.inf.br.
Obs.: É de responsabilidade do usuário o site escolhido para o redirecionamento.
Permite validar duplicidade de cnpj/cpf
Não permite que sejam incluídos dois Clientes/Unidades com o mesmo cnpj/cpf. Quando o cnpj/cpf preenchido já estiver cadastrado no sistema, não será permitido gravar.
Obrigatório preenchimento e validação do cnpj/cpf de Cliente
Não permite que um cadastro de cliente seja gravado sem cnpj/cpfou inválido.
Obrigatório preenchimento do Telefone 1
Quando selecionado, não permite gravar o cadastro do cliente sem o campo "Telefone 1" preenchido.
Obrigatório preenchimento do E-mail
Quando selecionado, não permite gravar o cadastro do cliente sem o campo "E-mail" preenchido.
Obrigatório preenchimento e validação do cnpj/cpf de Unidade
Não permite que um cadastro de unidade seja gravado sem cnpj/cpf ou inválido.
Obs.: Os Clientes/Unidades já cadastrados com ou sem cnpj/cpf, permanecerão da mesma forma. Os parâmetros acima só valerão para os mesmos, se o cnpj/cpf for alterado.
Valida Processos em Andamento
Quando selecionado, se houver um processo em andamento não será possível fazer alterações na tela de Mapear Processos.


Descrição dos campos

Código da Divisão Cliente Padrão
Permite selecionar uma Divisão no combo "Divisão Padrão para Cliente". Essa divisão será utilizada quando na inclusão de um Cliente, o usuário que irá incluí-lo estiver relacionado a "Todas as Divisões" no combo Cliente da tela de Configuração Padrão por Divisão. Nesse caso a divisão que será relacionada automaticamente ao cliente será a divisão padrão para cliente.
Utiliza Login Automático
Quando selecionado, será possível aos usuários da empresa possuírem um Token que lhes permitirá acessar o sistema sem que seja necessário utilizar login e senha.
